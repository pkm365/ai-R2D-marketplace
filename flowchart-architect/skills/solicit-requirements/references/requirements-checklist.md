# Requirements Gathering Checklist

Use this checklist to ensure you have all the necessary information before starting the flowchart design.

## 1. Process Overview
- [ ] What is the exact name of the process?
- [ ] Who is the Process Owner? (Person accountable for the process)
- [ ] What is the trigger? (What event starts this process?)
- [ ] What is the desired outcome? (What is the final result?)

## 2. Stakeholders & Swimlanes
- [ ] Which departments are involved?
- [ ] Are there specific roles within those departments?
- [ ] Are there external entities (Customers, Suppliers, Government)?
- [ ] Which systems are touched? (ERP, CRM, WMS, MES)

## 3. Process Logic
- [ ] What is the very first step?
- [ ] What are the decision points? (Where does the path split?)
- [ ] What happens in the "Happy Path" (Success scenario)?
- [ ] What happens in the "Exception Path" (Failure/Rejection scenario)?
- [ ] Are there any parallel activities? (Things happening at the same time)

## 4. Inputs & Outputs
- [ ] What documents/data are required to start?
- [ ] What documents/data are generated during the process?
- [ ] Are there specific forms or templates used?

## 5. Business Rules
- [ ] Are there approval thresholds? (e.g., > $5000 needs Manager approval)
- [ ] Are there time constraints? (e.g., must be done within 24 hours)
- [ ] Are there compliance requirements? (ISO, FDA, SOX)
